Worklog này ghi lại hành trình thực tập trong 12 tuần, tập trung vào việc học các kiến thức nền tảng về AWS, thực hành vận hành cloud và tham gia các workshop thực tế. Mỗi tuần tóm tắt mục tiêu chính, các công việc đã thực hiện và những kết quả đạt được trong quá trình học tập và phát triển dự án.

Các phần worklog theo tuần dưới đây cung cấp một cái nhìn tổng quan có hệ thống về tiến độ đã đạt được, bắt đầu từ việc làm quen với AWS cho đến giai đoạn tổng hợp kiến thức, hoàn thiện gameplay online và chuẩn bị triển khai server game lên cloud:

* **Tuần 1:** [Làm quen với AWS và các dịch vụ cơ bản](1.1-Week1/)

* **Tuần 2:** [Làm quen với Unity Engine và workflow phát triển game cơ bản, tìm hiểu AWS IAM và quản lý phân quyền, chuẩn bị nền tảng để triển khai các hệ thống liên quan đến game trên AWS.](1.2-Week2/)

* **Tuần 3:** [Hiểu kiến trúc AWS VPC và workflow triển khai, học cách tạo và cấu hình EC2 instance, thực hành truy cập từ xa bằng SSH và cấu hình bảo mật, làm quen với việc quản lý tài nguyên compute trên cloud.](1.3-Week3/)

* **Tuần 4:** [Tìm hiểu workflow phát triển Unity nâng cao, hiểu hệ thống physics và animation trong Unity, cải thiện kiến trúc lập trình C# cho các dự án game có khả năng mở rộng, thực hành các kỹ thuật tối ưu hiệu năng cơ bản cho game.](1.4-Week4/)

* **Tuần 5:** [Tìm hiểu tổng quan về Amazon EC2 trên AWS, học cách triển khai và quản lý EC2 instance, thực hành cấu hình Security Group, Key Pair và SSH, làm quen với workflow quản trị Linux Server trên cloud, tìm hiểu monitoring và quản lý tài nguyên EC2 cơ bản.](1.5-Week5/)

* **Tuần 6:** [Cập nhật worklog dựa trên tiến độ thực tế trong Unity, cải thiện cấu trúc project và các hệ thống gameplay hiện có, tiếp tục rà soát Player, Enemy, FSM, Core System, Animation và Combat logic, tracking tiến độ từ thứ Hai trong tuần, tham dự sự kiện cuối tuần và ghi nhận các nội dung học được.](1.6-Week6/)

* **Tuần 7:** [Tiếp tục cải thiện hệ thống combat và knockback trong project Unity, khắc phục lỗi Player bị nhận knockback liên tục và không thể điều khiển trở lại, xây dựng nền tảng cho Boss Minotaur bằng FSM và Core System hiện có, thiết lập các trạng thái tấn công, phòng thủ, choáng, chuyển phase và tử vong của Boss, kiểm tra sự đồng bộ giữa gameplay logic, Rigidbody2D, Animator và Animation Events.](1.7-Week7/)

* **Tuần 8:** [Hoàn thành các lab AWS còn lại, tham dự sự kiện AWS và ghi nhận các kiến thức học được, rà soát các nội dung liên quan đến AWS IAM, CloudWatch, AWS CLI, AWS Budget, AWS Support, Resource Groups và các lab liên quan đến Lambda.](1.8-Week8/)

* **Tuần 9:** [Hoàn thiện nền tảng gameplay top-down cốt lõi cho dự án game 2D co-op boss fight, chuyển đổi các hệ thống cũ từ platformer sang top-down, đặc biệt là di chuyển, aiming, combat và weapons, xây dựng cơ chế Boss Minotaur với nhiều trạng thái, kỹ năng, knockback, block và parry, thực hiện các bài test multiplayer đầu tiên bằng Unity Netcode for GameObjects.](1.9-Week9/)

* **Tuần 10:** [Hoàn thiện hệ thống vũ khí top-down với nhiều loại weapon khác nhau, bổ sung hiệu ứng hình ảnh cho đòn đánh, Block, Parry và projectile, tạo công cụ debug để hiển thị hitbox, hướng Block và hướng Parry, hoàn thành kiểm thử online phía Unity, chuẩn bị workflow triển khai AWS đầu tiên để build và host game server.](1.10-Week10/)

* **Tuần 11:** [Hoàn thiện các tính năng Online quan trọng cho game TopDown co-op boss fight, tích hợp luồng tạo phòng, tham gia phòng và chuyển scene giữa Lobby Scene, Map Lobby và Boss Room, phát triển Boss Phoenix theo phong cách Bullet Hell với nhiều trạng thái và kỹ năng, hoàn thiện hệ thống kết quả Boss Room bao gồm quay lại Map Lobby, kiểm tra toàn bộ Player đã chết và spawn phần thưởng sau khi đánh bại Boss, cải thiện hệ thống vũ khí, phần thưởng và kỹ năng hỗ trợ cho Player.](1.11-Week11/)

* **Tuần 12:** [Hoàn thiện kiểm thử gameplay online trong Unity trước khi chuyển sang triển khai online bên ngoài, chuẩn bị workflow triển khai AWS cho server multiplayer của RoughLife, build và cấu hình Unity Linux Dedicated Server để host trên cloud, thiết lập hướng kiến trúc AWS ban đầu sử dụng EC2, Lambda Function URL và DynamoDB, tổ chức lại tài liệu dự án, worklog và event report để chuẩn bị nộp báo cáo cuối cùng.](1.12-Week12/)
