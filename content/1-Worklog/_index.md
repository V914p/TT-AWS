This worklog documents a 12-week internship journey focused on learning AWS fundamentals, practicing cloud operations, and participating in hands-on workshops. Each week summarizes the main objective, the tasks completed, and the key outcomes achieved during the program.

The weekly sections below provide a structured overview of the progress made from the first introduction to AWS through the final consolidation of knowledge and practical experience:

- **Week 1:** [Getting familiar with AWS and basic services](1.1-Week1/)

- **Week 2:** [Get familiar with Unity Engine and the basic game development workflow, Understand AWS IAM and permission management, Prepare the foundation for deploying game-related systems on AWS.](1.2-Week2/)

- **Week 3:** [* Understand AWS VPC architecture and deployment workflow, Learn how to create and configure EC2 instances, Practice SSH remote access and security configuration, Become familiar with managing compute resources on the cloud.](1.3-Week3/)

- **Week 4:** [Learn advanced Unity development workflow. Understand Unity physics and animation systems. Improve C# scripting architecture for scalable game projects. Practice basic optimization techniques for game performance.](1.4-Week4/)

- **Week 5:** [Learn the overview of Amazon EC2 on AWS. Learn how to deploy and manage EC2 instances. Practice configuring Security Groups, Key Pairs, and SSH. Become familiar with Linux server administration workflow on the cloud. Learn basic EC2 monitoring and resource management.](1.5-Week5/)

- **Week 6:** [Update the worklog based on the actual Unity development progress. Improve the project structure and existing gameplay systems. Continue reviewing Player, Enemy, FSM, Core System, Animation, and Combat logic. Track progress from Monday of this week. Attend the weekend event and record useful takeaways.](1.6-Week6/)

- **Week 7:** [Continue improving the combat and knockback systems in the Unity project. Fix the issue where Player receives continuous knockback and cannot regain control. Build the foundation for the Minotaur Boss using the existing FSM and Core System. Set up Boss attack, defense, stun, phase transition, and death states. Test synchronization between gameplay logic, Rigidbody2D, Animator, and Animation Events.](1.7-Week7/)

- **Week 8:** [Complete the remaining AWS labs. Attend the AWS event and record useful learning outcomes. Review AWS IAM, CloudWatch, AWS CLI, AWS Budget, Support, Resource Groups, and Lambda-related labs.](1.8-Week8/)

- **Week 9:** [Complete the core top-down gameplay foundation for the 2D co-op boss fight project. Convert legacy platformer systems into top-down systems, especially movement, aiming, combat, and weapons. Build the Minotaur boss mechanics with multiple states, skills, knockback, block, and parry interactions. Run the first multiplayer tests using Unity Netcode for GameObjects.](1.9-Week9/)

- **Week 10:** [Complete the top-down weapon system with multiple weapon types. Add visual effects for attacks, Block, Parry, and projectiles. Create debug tools to visualize hitboxes, Block direction, and Parry direction. Complete Unity-side online testing. Prepare the first AWS deployment workflow for building and hosting the game server.](1.10-Week10/)

- **Week 11:** [Complete important Online features for the TopDown co-op boss fight game. Integrate the room creation, room joining, and scene transition flow between Lobby Scene, Map Lobby, and Boss Room. Develop a new Bullet Hell style Phoenix Boss with multiple states and skills. Complete the Boss Room result system, including returning to Map Lobby, checking if all players are dead, and spawning rewards after defeating the Boss. Improve the weapon, reward, and support skill systems for the Player.](1.11-Week11/)

- **Week 12:** [ Finalize the Unity-side online gameplay testing before moving to external online deployment. Prepare the AWS deployment workflow for the RoughLife multiplayer game server. Build and configure the Unity Linux Dedicated Server for cloud hosting. Set up the first AWS architecture direction using EC2, Lambda Function URL, and DynamoDB. Organize project documentation, worklogs, and event reports for final submission.](1.12-Week12/)
